# 边与设计原则

## 边（Edge）行为规则

### 创建阶段（edge:add）

| 规则 | 说明 |
|--|--|
| 禁止重复连线 | 同一 source → target 仅允许一条 |
| 类型匹配 | source / target 类型必须符合节点定义 |

### 全流程阶段（validateGraph）

| 规则 | 说明 |
|--|--|
| 条件完整性 | 排他 / 包容网关的条件校验 |
| 默认连线 | 排他网关默认线校验 |
| 并行约束 | 并行网关不允许条件 |

## 错误级别（扩展）

| 级别 | 含义 |
|--|--|
| ERROR | 阻止发布 |
| WARN | 允许发布，但提示 |

## 设计原则总结

- **连线创建 ≠ 流程完成**
- **创建阶段只做结构合法性校验**
- **发布阶段才做完整性校验**
- 所有行为规则集中在 `NodeBehavior`
- 所有执行逻辑集中在 `BehaviorsEngine`
